# 🐟 海鲜市集活鲜价牌与预订系统

多端适配的海鲜市集活鲜价牌与预订页面，同时适配档口大屏、顾客手机和员工平板。

## 原始需求

> 请制作海鲜市集活鲜价牌与预订页面，React 页面同时适配档口大屏、顾客手机和员工平板，展示活鲜池、今日价格、规格重量、加工方式、预订取货、排队号、售罄替代和称重记录。顾客查看波龙、梭子蟹、鲍鱼、石斑或皮皮虾时，要知道鲜活状态、池号、估重范围、加工费和预计取餐；档口员工更新换水、缺氧、称重、宰杀、蒸煮、打包和取餐进度。页面要有市集热闹感，但价牌数字、重量差额和制作状态必须比装饰更醒目，避免顾客以为预估价就是最终价。
>
> 扩展活鲜池状态同步：某个水池出现缺氧、临时换池、售罄或新货到场时，大屏价牌、手机预订和员工平板同时变化，顾客能看到替代品种、规格差异、价格浮动以及是否需要重新确认订单。
> 扩展称重补差流程：顾客按估重预订，到档口捞取称重后，页面展示实际重量、加工费、排队号、需补或应退金额；用户确认前不能进入制作，员工修改重量时必须保留前后数值和原因。

## 功能概览

### 活鲜池展示
- 波龙、梭子蟹、鲍鱼、石斑、皮皮虾等活鲜卡片，含鲜活状态呼吸灯、池号、今日价格、估重范围
- 预估价≠最终价醒目提示：价格区域红色警告"以实际称重为准"，重量差额范围展示

### 活鲜池状态同步
- **缺氧状态**：某水池缺氧时，大屏、手机、员工端同步显示缺氧警告标识和事件消息
- **临时换池**：支持临时换池操作，显示原池号和新池号，以及换池原因
- **售罄状态**：售罄品种灰显，自动推荐替代品种，显示规格差异和价格浮动
- **新货到场**：新货标识闪烁提示，显示到货时间
- **订单重确认提示**：状态变化可能影响已预订订单时，提示需重新确认

### 称重补差流程
- **实际重量展示**：显示预估重量 vs 实际称重对比
- **加工费明细**：清晰展示单价和加工费组成
- **排队号**：醒目显示订单排号
- **需补/应退金额**：差额超过阈值时自动计算补差金额，需补差价（橙色）或应退金额（绿色）
- **顾客确认机制**：称重后需顾客确认才能进入制作环节
- **重量修改记录**：员工修改重量时必须填写原因，系统自动保存修改历史（前后数值、原因、操作人、时间）

### 加工与取餐
- 加工方式选择：清蒸、蒜蓉、椒盐、刺身等，标注加工费和预计时间
- 预订与排队：提交预订获取排队号，五步加工进度追踪
- 称重记录：实际 vs 预估对比，差额醒目标注
- 售罄替代：售罄品种灰显并推荐替代品

### 多端适配
- **员工管理面板**：换水、缺氧、称重录入、加工进度更新、售罄管理、临时换池、新货登记、价格调整
- **大屏价牌模式**：全屏价牌展示，特大字价格，可取餐排号滚动
- **顾客手机端**：预订、订单追踪、称重确认

## 技术栈

- React 18 + TypeScript + Vite
- Tailwind CSS 3
- Zustand 状态管理（支持 localStorage 持久化和多标签同步）
- React Router DOM 7
- Lucide React 图标
- 纯前端，Mock 数据

## 启动方式

### 前置要求

- Node.js 18+
- npm 或 pnpm

### 启动步骤

#### 1. 安装依赖

```bash
npm install
```

#### 2. 启动服务

```bash
npm run dev
```

访问地址：http://localhost:5173

### Docker 启动方式

#### 前置要求

- Docker 20+
- Docker Compose V2

#### 构建并启动

```bash
docker compose up --build
```

后台运行：

```bash
docker compose up --build -d
```

停止并清理：

```bash
docker compose down
```

访问地址：http://localhost:3000

## 页面路由

| 路由 | 页面 | 说明 |
|------|------|------|
| `/` | 活鲜池首页 | 展示所有活鲜卡片，自动适配设备 |
| `/seafood/:id` | 活鲜详情页 | 价格、加工方式、预订下单、替代品种 |
| `/order/:id` | 订单追踪页 | 排队号、加工进度、称重记录、补差确认 |
| `/staff` | 员工管理面板 | 池状态、称重录入、进度更新、修改记录 |
| `/board` | 大屏价牌模式 | 全屏价牌，适合档口大屏展示 |

## 设计说明

- 深海蓝底色营造海洋氛围，珊瑚橙价格醒目温暖
- 鲜活状态呼吸灯动画：绿色=鲜活，黄色=活力不足，红色闪烁=缺氧
- 价格数字使用等宽字体(JetBrains Mono)，确保数字对齐醒目
- 预估价区域始终标注红色"以实际称重为准"警告，重量差额范围清晰展示
- 状态事件徽章：新货（绿色闪烁）、缺氧（橙色闪烁）、换池（蓝色）、售罄（灰色）
- 补差金额：需补差价橙色警示，应退金额绿色提示
- 称重修改历史可展开查看，保留前后数值、原因和操作人

## 核心状态字段说明

### 海鲜状态（SeafoodItem）

| 字段 | 说明 |
|------|------|
| `poolEvent` | 池子事件类型：low_oxygen / temporary_transfer / sold_out / new_arrival / normal |
| `isLowOxygen` | 是否缺氧 |
| `isTemporaryTransferred` | 是否临时换池 |
| `isNewArrival` | 是否新货到 |
| `isSoldOut` | 是否售罄 |
| `needsOrderReconfirm` | 是否需要订单重确认 |
| `substituteId` | 替代品种ID |
| `originalPoolNumber` | 原始池号 |
| `originalPricePerJin` | 原始价格 |

### 订单状态（Order）

| 字段 | 说明 |
|------|------|
| `status` | 订单状态：pending / waiting_confirm / processing / ready / completed / cancelled |
| `weightConfirmed` | 重量是否已确认 |
| `needsCustomerConfirm` | 是否需要顾客确认 |
| `supplementAmount` | 需补金额 |
| `refundAmount` | 应退金额 |
| `weightModifications` | 重量修改记录数组 |
