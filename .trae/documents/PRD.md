# 海鲜市集活鲜价牌与预订系统 - 产品需求文档

## 1. 产品概述
海鲜市集活鲜价牌与预订系统是一个多端适配的 React 应用，同时服务于档口大屏、顾客手机和员工平板三大场景。系统核心解决活鲜交易中价格透明、重量预估偏差、加工流程追踪和排队取餐等痛点，确保顾客始终清楚预估价≠最终价。

- 目标用户：海鲜市集顾客、档口员工、档口老板
- 核心价值：让活鲜交易信息透明化，减少因价格和重量误解产生的纠纷，提升取餐效率

## 2. 核心功能

### 2.1 用户角色
| 角色 | 进入方式 | 核心权限 |
|------|----------|----------|
| 顾客 | 扫码/直接访问 | 浏览活鲜、查看价格、预订、查看取餐进度 |
| 员工 | 扫码/密码进入 | 更新池状态、称重录入、更新加工进度、确认取餐 |
| 大屏模式 | 自动检测/手动切换 | 展示价牌、池号、排队号、售罄信息 |

### 2.2 功能模块

1. **活鲜池展示页**: 活鲜分类展示、池号、鲜活状态指示灯、今日价格、估重范围、售罄标记与替代推荐
2. **活鲜详情页**: 规格重量详情、加工方式与加工费、预估总价（醒目标注为预估）、预订下单
3. **预订与取餐页**: 排队号展示、加工进度追踪、称重记录、最终价格、取餐通知
4. **员工管理面板**: 换水/缺氧状态更新、称重录入、加工进度更新（宰杀→蒸煮→打包→取餐）、售罄设置

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 活鲜池展示页 | 活鲜卡片列表 | 展示波龙、梭子蟹、鲍鱼、石斑、皮皮虾等活鲜卡片，含鲜活状态灯、池号、今日单价、估重范围 |
| 活鲜池展示页 | 售罄与替代 | 售罄品种灰显并标注"售罄"，推荐同品类替代品 |
| 活鲜池展示页 | 大屏模式 | 自动切换为大字价牌，突出价格、池号、排队号滚动 |
| 活鲜详情页 | 规格与重量 | 展示规格等级、估重范围（如 1.2-1.5 斤）、按估重计算的预估价格 |
| 活鲜详情页 | 加工方式选择 | 清蒸、蒜蓉、椒盐、刺身等加工方式，每种标注加工费 |
| 活鲜详情页 | 价格醒目提示 | 预估总价用大字显示，紧邻"以实际称重为准"红色提示，重量差额范围说明 |
| 活鲜详情页 | 预订下单 | 选择数量、加工方式后提交预订，获取排队号 |
| 预订与取餐页 | 排队号展示 | 当前排队号、我的号码、预计等待时间 |
| 预订与取餐页 | 加工进度条 | 称重→宰杀→蒸煮→打包→取餐 五步进度 |
| 预订与取餐页 | 称重记录 | 实际称重重量、最终价格、与预估的差额 |
| 员工管理面板 | 池状态管理 | 更新换水时间、缺氧报警、活鲜存活状态 |
| 员工管理面板 | 称重录入 | 扫码/选择订单后录入实际重量，自动计算最终价格 |
| 员工管理面板 | 加工进度更新 | 逐项更新订单加工阶段 |
| 员工管理面板 | 售罄管理 | 标记品种售罄、设置替代推荐 |

## 3. 核心流程

### 3.1 顾客浏览与预订流程
顾客进入页面 → 浏览活鲜池（查看鲜活状态、价格、估重）→ 选择活鲜查看详情 → 了解预估价≠最终价提示 → 选择加工方式 → 提交预订获取排队号 → 等待加工进度更新 → 收到取餐通知 → 取餐时查看称重记录和最终价格

```mermaid
flowchart TD
    "顾客进入页面" --> "浏览活鲜池"
    "浏览活鲜池" --> "选择活鲜详情"
    "选择活鲜详情" --> "查看预估价提示"
    "查看预估价提示" --> "选择加工方式"
    "选择加工方式" --> "提交预订"
    "提交预订" --> "获取排队号"
    "获取排队号" --> "等待加工进度"
    "等待加工进度" --> "查看称重记录"
    "查看称重记录" --> "取餐确认"
```

### 3.2 员工操作流程
员工进入管理面板 → 查看待处理订单 → 称重录入 → 更新加工进度 → 通知取餐

```mermaid
flowchart TD
    "员工进入管理面板" --> "查看待处理订单"
    "查看待处理订单" --> "称重录入"
    "称重录入" --> "更新宰杀状态"
    "更新宰杀状态" --> "更新蒸煮状态"
    "更新蒸煮状态" --> "更新打包状态"
    "更新打包状态" --> "通知取餐"
    "通知取餐" --> "确认取餐完成"
```

## 4. 用户界面设计

### 4.1 设计风格
- **主题**: 港口市集风 - 热闹但不杂乱，信息优先
- **主色**: 深海蓝 (#0A2540) 作为底色，营造海洋氛围
- **辅色**: 珊瑚橙 (#FF6B35) 用于价格和关键操作按钮，醒目且温暖
- **强调色**: 翡翠绿 (#00C48C) 表示鲜活/可用状态
- **警告色**: 琥珀红 (#FF3B30) 用于售罄、缺氧、价格提示
- **字体**: 价格数字使用等宽粗体，确保数字对齐和醒目；正文使用清晰无衬线体
- **布局**: 卡片式布局，大屏模式切换为价牌行列式
- **装饰**: 海浪纹理、气泡动画作为背景装饰，但绝不遮挡价格和状态信息
- **核心原则**: 价格数字、重量差额、制作状态 > 装饰元素

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI 要素 |
|----------|----------|---------|
| 活鲜池展示页 | 活鲜卡片 | 圆角卡片、鲜活状态呼吸灯、池号标签、大字价格、估重范围 |
| 活鲜池展示页 | 大屏价牌 | 全屏价牌布局、价格特大字、排队号滚动条、售罄灰显 |
| 活鲜详情页 | 价格区域 | 预估价超大字、红色"以实际称重为准"标注、重量差额范围 |
| 活鲜详情页 | 加工选择 | 卡片式单选、加工费标注、预计取餐时间 |
| 预订与取餐页 | 进度条 | 五步横向进度条、当前步骤高亮、预计剩余时间 |
| 预订与取餐页 | 称重记录 | 实际vs预估对比表、差额醒目标注 |
| 员工管理面板 | 订单列表 | 紧凑列表、快捷状态按钮、称重输入框 |

### 4.3 响应式设计
- **大屏模式 (≥1200px)**: 4-5列活鲜卡片 / 全屏价牌模式，排队号独立区域
- **平板模式 (768-1199px)**: 2-3列卡片，员工操作面板侧边栏
- **手机模式 (<768px)**: 单列卡片，底部Tab导航，关键信息置顶
- **触控优化**: 按钮最小 44px 触控区域，员工端操作按钮更大

### 4.4 动效设计
- 鲜活状态灯：绿色呼吸灯动画（存活），红色闪烁（缺氧）
- 卡片进入：交错淡入动画
- 排队号更新：数字翻转动画
- 进度条：平滑过渡
- 背景：缓慢移动的气泡/波浪装饰（低透明度，不干扰阅读）
